# Day 08

## 受講者が実装する場所

- `day08/app.py` の `run_graph()`

## 目的

- 失敗時に復帰できるフロー（リトライ/フォールバック）を実装できるようにする
- 無限ループを防ぐ上限設計を入れられるようにする

## ゴール（完了条件）

- 失敗ケースでリトライ/フォールバックが動く
- 上限到達時に無限ループせず終了する

## 機能要件

コマンド

- `python -m day08.app` で起動できること（モジュール名は固定）

必須オプション

- `--text`：ユーザー入力（文字列、必須）

任意オプション

- `--max-steps`：最大ステップ数（整数、デフォルト `10`、1〜50）
- `--max-retry`：最大リトライ回数（整数、デフォルト `1`、0〜5）

出力

- 正常時：標準出力に結果（文章）を出す
- 失敗時：標準エラーにエラーメッセージを出す

終了コード

- 正常終了：`0`
- 入力不備：`2`
- 実行失敗（上限到達含む）：`1`

受け入れ基準

- 失敗ケースでリトライ/フォールバックが動く
- 上限到達時に無限ループせず、明示的に終了する

## 実装タスク

- `day08/app.py` の `run_graph()` を実装し、機能要件を満たす
- 失敗パターンと復帰パスを最低1つ実装し、上限を反映する

## 実行方法

### 実行コマンド例

```bash
# 基本的な実行
python -m day08.app --text "〇〇について教えて"

# 最大ステップ数とリトライ回数を指定
python -m day08.app --text "〇〇について教えて" --max-steps 5 --max-retry 2
```

### 失敗ケースを再現する入力例

```bash
# JSONパース失敗を誘発する入力（リトライが動く）
python -m day08.app --text "不完全なJSONを返して"

# 検索ヒットなしを誘発する入力（フォールバックが動く）
python -m day08.app --text "存在しない情報について教えて"
```
