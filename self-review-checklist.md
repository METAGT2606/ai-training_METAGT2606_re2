# セルフレビュー・チェックリスト（受講者向け）

各日の課題は、PR提出前にこのチェックリストを必ず確認します。

---

## 1. 基本（全課題共通）

- [ ] READMEに実行手順がある（コマンドをコピペで実行できる）
- [ ] 依存関係/前提（Pythonバージョン、必要な環境変数）が明記されている
- [ ] 実行してクラッシュしない（最低1回は手元で実行した）
- [ ] 例外時に分かりやすいメッセージが出る（何を直せばよいか分かる）
- [ ] ログが最低限入っている（どこで失敗したか追える）

---

## 2. テスト項目（最小セット）

「自動テストがあること」が目的ではなく、**壊れ方を把握していること**が目的です。

- [ ] 正常系：想定入力で期待どおりに動く
- [ ] 異常系：入力が空/不正/不足でも安全に失敗する
- [ ] 境界：長文入力や想定外の文字（改行、記号）が入っても破綻しない
- [ ] 再実行：同じ入力を2回実行しても極端に挙動がブレない（許容範囲内）

---

## 3. LLM/Bedrockを使う課題の追加チェック

- [ ] モデル/リージョン/主要パラメータが設定として分かる（コード内に埋め込みすぎない）
- [ ] コスト意識：不要な再実行やループをしていない
- [ ] 失敗時の切り分けができる（認証/権限/ネットワーク/モデル設定）

---

## 4. 構造化出力（JSONなど）を扱う課題の追加チェック

- [ ] 必須キーが揃っていることを検証している
- [ ] パース失敗時のフォールバックがある（再生成/エラー表示/縮退動作）

---

## 5. Tool calling / エージェント課題の追加チェック

- [ ] 実行可能なツールが許可リストで制限されている
- [ ] ツール引数のバリデーションがある（想定外の引数で実行されない）
- [ ] 無限ループしない（最大ステップ/回数などの上限がある）
- [ ] 失敗時の復帰パスがある（リトライ/フォールバック/ユーザーへの確認）

---

## 6. RAG課題の追加チェック

- [ ] 出典（URL/章/引用）が回答に含まれる
- [ ] 検索ヒットがない場合の挙動が定義されている
- [ ] 公開データのライセンス/出典が分かる（READMEまたはdocsに記載）

---

## 7. PR本文テンプレ（コピペ用）

- 対応した課題：
- 実行手順：
- 期待する結果：
- セルフレビュー（チェックした項目）：
- 困っている点（あれば）：
