# Day 04

## 受講者が実装する場所

- `day04/app.py` の `run_chain()`

## 目的

- LangChainで処理を部品化してつなげる
- Tool callingの最小構成を作り、ツールの引数検証/制限を理解する

## ゴール（完了条件）

- LLMからツールが1回以上呼び出される
- 不正引数でツールが実行されない

## ツール仕様

- ツール名：
- 入力：
- 出力：
- 許可リスト/制限：

例（どれか1つでOK）

- 日付取得ツール（`today`）
- 簡易計算ツール（`add`/`multiply`）
- 固定データ検索（`lookup_company_policy` など、ローカル辞書から検索）

本研修テンプレでは、未経験者が迷わないよう、以下のいずれか1つを必須ツールとして実装してください。

- ツール名：`today`
  - 入力：なし
  - 出力：`YYYY-MM-DD`（文字列）

または

- ツール名：`add`
  - 入力：`a`（整数）, `b`（整数）
  - 出力：`a + b`（整数）

## 機能要件

コマンド

- `python -m day04.app` で起動できること（モジュール名は固定）

必須オプション

- `--text`：ユーザー入力（文字列、必須）

出力

- 標準出力に最終回答（文章）を出力する

終了コード

- 正常終了：`0`
- 入力不備：`2`
- 実行失敗（LLM/ツール呼び出し失敗等）：`1`

必須要件

- 上記の必須ツール（`today` または `add`）を実装し、LLMから1回以上呼び出す
- ツール引数を検証する（`add`の場合は整数のみ許可、など）
- ツール呼び出しに失敗した場合に、安全に失敗する（終了コード `1` でよい）

受け入れ基準

- ツールが呼び出されたことがログで分かる
- 不正な引数でツールが実行されない

## 実装タスク

- `day04/app.py` の `run_chain()` を実装し、機能要件を満たす
- ツール（`today` または `add`）を実装し、ログで呼び出しが分かるようにする

## 実行方法

### 実行コマンド例

```bash
# todayツールを使う例
python -m day04.app --text "今日の日付を教えて"

# addツールを使う例
python -m day04.app --text "123と456を足した結果を教えて"
```

### 期待する出力例

```
# todayツールの場合
今日は2024-12-01です。

# addツールの場合
123と456を足すと579になります。
```
