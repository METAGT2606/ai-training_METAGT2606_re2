# GitHub運用ガイド（受講者向け）

この研修では、運営が提供するテンプレートリポジトリから各自リポジトリを作成し、課題を提出します。

---

## 1. 全体方針

- このリポジトリはテンプレートとして配布されます
- 受講者はテンプレートから自分用のリポジトリを作成し、以後は自分のリポジトリ上で作業します
- 受講者のリポジトリはPrivateで作成します

---

## 2. 受講者：リポジトリ作成手順

1. テンプレートリポジトリ画面で `Use this template` を選びます
2. `Create a new repository` を選びます
3. リポジトリ名を設定します（命名規則に従う）
4. Visibilityは `Private` を選びます
5. 作成後、講師/メンターをCollaboratorとして追加します（指示がある場合のみ）

### 2.1 リポジトリ命名規則

- 社員IDは含めない
- 受講者のフルネームを含める
- 半角英数字とハイフンを推奨する

例：

- `ai-training-taro-yamada`
- `ai-agent-training-hanako-suzuki`

---

## 3. 受講者：提出フロー（Pull Request）

課題はブランチを切ってPull Requestで提出します。

- `main` ブランチに直接pushしない
- 課題ごとにブランチを作成する
- Pull Requestを作り、レビュー/指摘を受けたら修正を追加pushする

提出前に、`self-review-checklist.md` を使ってセルフレビューを行います。

### 3.1 ブランチ命名（例）

- `feat/day01-setup`
- `feat/day03-prompt`
- `feat/final-agent`

### 3.2 Pull Requestテンプレ（本文に含める項目）

- 対応した課題：
- 実行手順：
- 期待する結果：
- 確認したこと（テスト/手動確認）：
- セルフレビュー（`self-review-checklist.md`でチェックした項目）：
- 困っている点（あれば）：

---

## 4. 提出物（成果物）の基本ルール

- 各課題は指定ディレクトリ配下に格納します（研修側の指示に従う）
- 実行方法はREADMEに追記します
- 成果物は第三者が再現できる状態にします
  - Pythonバージョン
  - 依存関係
  - 実行コマンド

---

## 5. 秘密情報・機密情報の取り扱い（最重要）

- APIキー、AWS認証情報、個人情報、社内情報はコミットしません
- `.env` などローカル設定ファイルは原則コミットしません
- 設定値は環境変数で扱います
- ログやスクリーンショットにも秘密情報を含めないよう注意します

---

## 6. AWS Bedrock利用に関する注意

- 利用するAWS認証方式（SSO / AssumeRole など）は研修の指示に従います
- コストに注意し、不要な連続実行や無限ループを避けます
- エラーが出た場合は、エラーメッセージと再現手順をPull Requestに記載します

---

## 7. よくあるトラブル

- `main` に直接pushしてしまった
  - 以後はブランチ運用に切り替え、必要なら講師に相談してください
- 認証情報をコミットしてしまった
  - すぐに講師へ連絡し、認証情報の無効化/ローテーションを行ってください
  - コミットの履歴からの削除が必要になる場合があります
