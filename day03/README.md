# Day 03

## 受講者が実装する場所

- `day03/app.py` の `generate_json()`

## 目的

- LLM出力を構造化（JSON）し、後段処理しやすい形にする
- JSONが壊れたときに安全に復帰/失敗できるようにする

## ゴール（完了条件）

- 標準出力にJSONのみが出力される
- JSONが壊れた場合に `--max-retry` の範囲で復帰できる（または明示的に失敗できる）

## JSON仕様

例として、次のようなJSONを返すタスクにします（必要に応じて調整してOK）。

- `title`：依頼の要約タイトル
- `tasks`：タスク配列
  - `id`：連番
  - `description`：作業内容
  - `acceptance_criteria`：完了条件
- `risks`：想定リスク

## 機能要件

コマンド

- `python -m day03.app` で起動できること（モジュール名は固定）

必須オプション

- `--requirements`：要件（文字列、必須）

任意オプション

- `--max-retry`：JSON破損時の再生成回数（整数、デフォルト `1`、0〜3）

出力

- 正常時：標準出力にJSONのみを出力する（余計な説明文を混ぜない）
- 失敗時：標準エラーにエラーメッセージを出力する

終了コード

- 正常終了：`0`
- 入力不備（未指定、空など）：`2`
- JSON生成/検証に失敗：`1`

検証

- 生成された文字列をJSONとしてパースできること
- `title` / `tasks` / `risks` のキーが存在すること
- `tasks` は配列であること

フォールバック

- JSONのパース/検証に失敗した場合、`--max-retry` の回数だけ再生成する
- それでも失敗した場合は終了コード `1` で終了する

## 実装タスク

- `day03/app.py` の `generate_json()` を実装し、機能要件を満たす
- JSON仕様に沿ったキー/型を安定して生成できるようにする

## 実行方法

### 実行コマンド例

```bash
# 基本的な実行
python -m day03.app --requirements "ユーザー登録機能を作る"

# リトライ回数を増やす
python -m day03.app --requirements "ECサイトの購入フローを設計する" --max-retry 3
```

### 期待する出力例

```json
{
  "title": "ユーザー登録機能の実装",
  "tasks": [
    {
      "id": 1,
      "description": "登録フォームのUIを作成する",
      "acceptance_criteria": "名前、メール、パスワードの入力欄がある"
    },
    {
      "id": 2,
      "description": "バリデーションを実装する",
      "acceptance_criteria": "必須項目の未入力時にエラーが表示される"
    }
  ],
  "risks": ["メールの重複チェックが必要", "パスワードの強度要件を決める必要がある"]
}
```
