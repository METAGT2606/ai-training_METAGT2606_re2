## 目的

- 最低限の評価（テスト入力セット）を作り、改善前後の差分を説明できるようにする

## ゴール（完了条件）

- テスト入力セットを `--suite` で実行できる
- 結果JSONを `--out` に出力できる

## 機能要件

コマンド

- `python -m day09.app` で起動できること（モジュール名は固定）

必須オプション

- `--suite`：テスト入力セットのパス（必須）

任意オプション

- `--out`：評価結果の出力パス（デフォルト `day09/result.json`）
- `--timeout-sec`：1ケースあたりのタイムアウト（整数、デフォルト `30`）

出力

- 正常時：標準出力にサマリ（成功/失敗件数）を出す
- 失敗時：標準エラーにエラーメッセージを出す

終了コード

- 正常終了：`0`
- 入力不備（`--suite` 不正など）：`2`
- 実行失敗：`1`

受け入れ基準

- `--suite` で指定したケースを順に実行できる（まだ中身はスタブでもよい）
- `--out` にJSONを出力する（成功/失敗の一覧を含める）

## 実行方法

### テスト入力セットの実行方法

```bash
# 基本的な実行
python -m day09.app --suite day09/suite.json

# 出力先を指定
python -m day09.app --suite day09/suite.json --out day09/result.json

# タイムアウトを指定
python -m day09.app --suite day09/suite.json --timeout-sec 60
```

### 結果の見方

- 標準出力：成功/失敗件数のサマリ
- `--out` で指定したファイル（デフォルト: `day09/result.json`）：詳細な結果JSON

