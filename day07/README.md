# Day 07

## 受講者が実装する場所

- `day07/app.py` の `run_graph()`

## 目的

- LangGraphで状態遷移を持つ処理フローを実装できるようにする
- stateに途中結果を保持し、デバッグ/説明ができるようにする

## ゴール（完了条件）

- 分岐が最低2パターン発生する
- どの分岐に入ったかがログで分かる

## 機能要件

コマンド

- `python -m day07.app` で起動できること（モジュール名は固定）

必須オプション

- `--text`：ユーザー入力（文字列、必須）

任意オプション

- `--mode`：分類方式（`llm` / `rule`、デフォルト `rule`）

出力

- 正常時：標準出力に結果（文章）を出す
- 失敗時：標準エラーにエラーメッセージを出す

終了コード

- 正常終了：`0`
- 入力不備：`2`
- 実行失敗：`1`

受け入れ基準

- 入力に応じて分岐が最低2パターン発生する
- どの分岐に入ったかがログで分かる
- 分類不能時はデフォルト分岐へ入る（または明示的に失敗する）

## 実装タスク

- `day07/app.py` の `run_graph()` を実装し、機能要件を満たす
- stateに途中結果を保持し、分岐の根拠が追えるようにする

## 実行方法

### 実行コマンド例

```bash

# LLMベースの分類

```

### 実装内容（例）

`day07/` 配下に、次のLangGraphを実装します。

- ノード1：入力を分類する（intentを決める）
